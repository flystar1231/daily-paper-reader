## 摘要
大规模文本到图像模型因其RGB生成预训练能学习丰富的语义、结构和几何先验，成为密集预测任务有吸引力的骨干网络。现有的生成和编辑方法通过将密集预测视为目标生成来复用这些先验：深度、法线、Alpha抠图、掩膜和热力图等标注被编码到RGB训练的VAE潜在空间中，再解码为类图像目标。我们认为这种方式继承了更多生成式输出接口，超出了密集预测所需：与RGB合成不同，密集预测要求在相同图像平面上获得像素级正确的、任务原生的场，而非渲染新的RGB内容。我们的关键观察是，预训练的DiT已经通过图像平面上的“块到令牌到块”网格组织RGB输入，因此每个令牌索引一个固定输出块，其通道可携带任务原生量而非RGB外观。我们将其实例化为ReChannel：保留DiT输入分布的VAE编码器，但丢弃目标侧解码器，使用任务LoRA适配冻结的DiT，并通过共享的令牌局部线性头（约33K参数，无空间混合）将每个令牌映射到其p x p x K_t像素空间块。使用FLUX-Klein，我们在六个密集预测任务和十多个基准上进行了评估。这种极简接口在无三图抠图、KITTI深度和指代分割上取得了新的最优结果，并在法线、显著性和姿态上保持竞争力。在匹配的4B参数设置下，它比“编辑+潜在解码”对应方法更准确且快2.48倍——密集感知可以从生成式预训练中受益，而无需继承其输出接口。

## Abstract
Large-scale text-to-image models are attractive backbones for dense prediction because RGB generation pretraining learns rich semantic, structural, and geometric priors. Existing generative and editing approaches reuse these priors by casting dense prediction as target generation: annotations such as depth, normals, alpha mattes, masks, and heatmaps are encoded into an RGB-trained VAE latent space and decoded back as image-like targets. We argue this inherits more of the generative output interface than dense prediction requires: unlike RGB synthesis, dense prediction asks for pixel-correct, task-native fields on the same image plane, not new RGB content to be rendered. Our key observation is that a pretrained DiT already organizes RGB inputs through a patch-to-token-to-patch lattice on the image plane, so each token indexes a fixed output patch whose channels can carry task-native quantities instead of RGB appearance. We instantiate this as ReChannel: we keep the VAE encoder for the DiT's input distribution but drop the target-side decoder, adapt the frozen DiT with task LoRA, and map each token to its p x p x K_t pixel-space patch through a shared token-local linear head--about 33K parameters, no spatial mixing. Using FLUX-Klein, we evaluate on six dense prediction tasks and over a dozen benchmarks. This minimal interface sets new state-of-the-art on trimap-free matting, KITTI depth, and referring segmentation, and stays competitive on normals, saliency, and pose. In a matched 4B setting it is more accurate and 2.48x faster than an edit-plus-latent-decode counterpart--dense perception can benefit from generative pretraining without inheriting its output interface.

## 论文核心总结：ReChannel——从生成到读出

### 一、 论文的核心问题与整体含义（研究动机和背景）

*   **核心问题**：当前基于文本到图像（T2I）生成模型的密集预测方法，存在**输出接口不匹配**的问题。这些方法将深度、法线、分割掩膜等密集预测目标视为一种“图像”来生成，需要将目标编码到RGB训练好的VAE潜在空间中，再解码回像素空间。这种“生成”范式继承了生成模型的输出接口，引入了不必要的计算开销和潜在的性能偏差。论文的核心问题就是：**密集预测是否必须继承这种生成式的输出接口？**
*   **整体含义**：论文认为，密集预测与RGB生成任务本质上是**不对称**的。RGB生成需要重建复杂的色彩、纹理等外观信息，而密集预测只需要在同一个图像平面上输出任务原生的、像素级别的场（如深度、法线）。因此，预训练生成模型提供的应该是强大的**图像形成先验**，而不是必须通过一个RGB渲染路径来输出结果。论文的核心贡献在于**将密集预测从“目标生成”重新框定为“场读出”**。

### 二、 论文提出的方法论：核心思想、关键技术细节

*   **核心思想**：**ReChannelling（信道重定向）**。利用预训练DiT（Diffusion Transformer）固有的“**图像块→令牌→图像块**”的空间网格结构。在这个结构中，每个输出图像块都是一个空间载体，其通道在生成任务中代表RGB外观，但在密集预测任务中可以被重新定向为携带任务原生的数量（如深度值）。
*   **关键技术细节**：
    1.  **保留输入接口，丢弃输出接口**：保留T2I模型中的VAE编码器，以确保输入分布与预训练一致。**明确丢弃目标侧的VAE解码器**，因为目标的最终监督和评估都在像素空间中进行。
    2.  **冻结骨干网络与轻量级适配**：冻结预训练的DiT骨干网络，仅训练任务特定的LoRA（Low-Rank Adaptation）参数来微调特征场，使之对齐目标语义。
    3.  **令牌局部线性读出**（Token-Local Linear Readout）：这是最核心的设计。经过LoRA适配后，每个空间令牌（$z_{ij}^t$）通过一个**共享的、仅约33K参数的线性投影层**（$W^t$ 和 $b^t$），直接映射到其在图像平面上对应位置的 $p \times p \times K_t$ 大小的像素空间目标块（$\hat{Y}_{ij}^t$）。这个投影层不包含任何空间混合操作，输出目标的空间结构完全由适配后的令牌场自身提供。
    4.  **公式化表达**：
        *   $Z_t = F_{\theta + \Delta t}(\text{VAE}_{\text{enc}}(x); \sigma=0, c_t)$：DiT骨干网络在零噪声条件下，基于LoRA适配，将VAE编码的图像输入组织成空间令牌场 $Z_t$。
        *   $\hat{Y}_{ij}^t = \text{reshape}(W^t z_{ij}^t + b^t)$：通过共享的线性头，将每个令牌直接读出为像素空间的目标块。

### 六、 论文的主要结论与发现

1.  **输出接口是关键瓶颈**：当前生成式密集预测方法中，目标侧VAE解码器是多余的，它不仅带来了计算开销，还可能限制性能。
2.  **ReChannel的有效性**：通过将DiT令牌场的通道“重新解释”为任务原生场，一个简单的令牌局部线性读出器就足以处理多种密集预测任务。
3.  **达到或超越SOTA**：在**无trimap抠图、KITTI深度、指代分割**任务上取得新的最优结果，并在法线、显著性、姿态上具有竞争力。
4.  **效率与准确性兼得**：在匹配的4B参数设置下，ReChannel比“编辑+潜在解码”方法准确率更高，且**快2.48倍**，证明其设计并非以牺牲性能为代价换取速度，而是二者双赢。

### 八、 不足与局限

1.  **骨干网络依赖性**：实验主要基于FLUX-Klein系列模型。方法的通用性和最优性是否在所有T2I模型上成立，尚需进一步验证。论文在结论中也将“将ReChannel扩展到FLUX-Klein之外”列为未来工作。

5.  **开放性问题**：该方法本质上是确定性的（零噪声）。论文没有深入探讨引入随机性（如多次采样求平均）是否能进一步提升性能，这可能是一个潜在的改进方向。
