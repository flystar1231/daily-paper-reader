## 摘要
AI生成的视频（AIGV）通常包含由帧间不一致而非单帧内引起的细微时间伪影。因此，检测此类伪影的检测器应受益于视频预训练骨干网络而非仅图像预训练骨干网络。然而，在实践中，采用标准全局读出的视频骨干网络往往无法在AIGV基准测试上优于强大的图像预训练探针。我们将这一差距归因于读出中的过度时空聚合。视频预训练骨干网络倾向于将每一帧压缩为单个全局描述符。这种压缩抑制了局部块级时间动态，并丢弃了块间关系，而正是这些线索是AIGV检测最可靠依赖的。基于此，我们提出了速度门控块速度剖绘（V-PVP），这是一种轻量级读出模块，仅将聚合层替换为块速度场上的两个并行流，仅增加约0.5M可训练参数。V-PVP作为通用的即插即用模块，在端到端微调和线性探测设置下，均能持续提升各种视频骨干网络的性能。我们的方法在AIGVDBench上达到95.28 AUC，同时保持骨干网络完全冻结。结果表明，仅替换聚合层就能重新激活冻结视频骨干网络的时间潜力，恢复其在AIGV检测上的优势。代码可在https://anonymous.4open.science/r/PVP-81B3/获取。

## Abstract
AI-generated videos (AIGVs) typically contain subtle temporal artifacts that arise from inter-frame inconsistencies rather than within individual frames. A detector that captures such artifacts should therefore benefit from video pretrained backbones over image only ones. In practice, however, video backbones with standard global readouts often fail to outperform strong image pretrained probes on AIGV benchmarks. We attribute this gap to excessive spatiotemporal aggregation in the readout. Video pretrained backbones tend to compress each frame into a single global descriptor. This compression suppresses local patch level temporal dynamics and discards inter patch relations, which are precisely the cues that AIGV detection most reliably depends on. Based on this, we propose Velocity Gated Patch Velocity Profiling (V-PVP), a lightweight readout that replaces only the aggregation layer with two parallel streams over the patch velocity field, adding only about $0.5$M trainable parameters. V-PVP serves as a general plug-and-play module that consistently improves performance across diverse video backbones under both end-to-end fine-tuning and linear probing settings. Our method reaches \textbf{95.28} AUC on AIGVDBench while keeping the backbone fully frozen. The results show that simply replacing the aggregation layer reactivates the temporal potential of frozen video backbones, restoring their advantage on AIGV detection. Code is available at https://anonymous.4open.science/r/PVP-81B3/.

- 核心动机：识别并修复视频骨干在 AIGV 检测中的“读出瓶颈”，仅通过替换聚合层即可重新激活冻结骨干的时序潜力，从而在极轻量参数下超越全微调方案。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 提出 **Velocity Gated Patch Velocity Profiling (V-PVP)**，一种轻量读出模块，**保持骨干完全冻结**，仅替换最终的聚合层（GAP/CLS）。模块基于 patch 速度场（相邻帧之间的 patch 令牌差值）构建两个并行流，分别解决论文诊断出的两类信息丢失：
  - **Loss 1（跨 patch 关系丢失）**：标准聚合抹去了不同 patch 之间的协变结构。V-PVP 使用速度门控可学习聚合，通过一个小型 MLP 为每个 patch 生成输入自适应的注意力权重（基于 patch 令牌本身及其速度幅值），从而在聚合后仍保留跨 patch 的异质性信息。
  - **Loss 2（每通道速度幅度稀释）**：取平均时不同 patch 的速度向量可能方向相反导致幅度抵消。V-PVP 使用通道忠实速度流，对每个 patch 的速度逐元素取绝对值后再平均，从而保留每通道的速度幅度信息。
- 两个流分别通过独立线性投影映射到 D/2 维，沿通道拼接，再送入一个浅层时间卷积头（Conv1D, kernel size 3）得到最终二分类 logit。
- 总可训练参数仅约 **0.5M**（占 VideoMAE-Base 的 0.57%），骨干完全冻结。

### 关键技术细节
- **patch 速度**：\( v_t^{(p)} = z_{t+1}^{(p)} - z_t^{(p)} \)（相邻帧相同空间位置的 patch 令牌差）。
- **速度门控聚合**：\( \alpha_t^{(p)} = \text{softmax}_p(\text{MLP}([z_t^{(p)}, \|v_t^{(p)}\|_2])), \quad \tilde{z}_t = \sum_p \alpha_t^{(p)} z_t^{(p)} \)。
- **通道忠实流**：\( r_t = \frac{1}{P} \sum_p |v_t^{(p)}| \)。
- **双流融合**：\( x_t = [W_z \tilde{z}_t; W_r r_t] \) 得到每个时间步的 D 维特征，经过 Conv1D + GELU + MeanPool + Dropout + Linear 得到输出。
- 训练采用二分类交叉熵损失，类别权重平衡真实/伪造样本不均衡，AdamW 优化器，学习率 1e-3，余弦退火，batch size 192，混合精度。

## 6. 论文的主要结论与发现

- **读出层是视频骨干在 AIGV 检测中的主要瓶颈**：标准全局聚合（GAP/CLS）同时做时空和语义坍塌，抑制了 patch 级时序变化和跨 patch 结构，而这些正是 AIGV 检测最需要的信号。
- **V-PVP 仅替换聚合层即可显著提升性能**：在 AIGVDBench 上，冻结 VideoMAE + V-PVP 达到 **95.28 AUC**，比全微调的标准读出（88.34）高 6.94 点，同时仅训练 0.5M 参数（全微调的 0.57%）。在 GenVidBench-143k 上也排名第一（93.75 AUC）。
- **冻结骨干 + V-PVP 甚至优于全微调的标准读出**（88.34 vs 95.28），表明投资容量应放在读出层而非骨干本身。
- **V-PVP 与骨干适应（LoRA/全微调）兼容且进一步提升**，最高达 +8.9 AUC（全微调场景）。
- **跨不同类型骨干（ViT、CNN）均有效**，但 CNN 上增益较小（约 +2.8），与诊断一致（卷积已先期部分聚合）。

## 7. 优点

- **方法设计精巧**：从诊断（两种信息丢失）出发，设计两个并行流分别修复，各流均有清晰物理意义和数学推导（附录 D）。速度门控的关键在于将注意力锚定在生成器无关的物理量 \(\|v_t^{(p)}\|_2\) 上，防止过拟合训练生成器的空间指纹。
- **极轻量与效率**：仅 0.5M 可训练参数，骨干完全冻结，训练成本极低（3–8 GPU 小时/配置），易于部署和重新校准。
- **通用性**：即插即用，跨多种视频骨干（TimeSformer、VideoMAE、UniFormerV2、Qwen3-VL-8B、I3D、SlowFast）和多种适应方案（冻结、LoRA、全微调）均出一致提升。
- **实验充分严谨**：包含跨生成器泛化、消融、骨干对比、适应方案组合、CNN 验证、诊断统计等多个维度；采用固定种子、多次运行和标准统计指标。
- **开源代码**：提供代码和预训练权重，便于复现和扩展。

## 8. 不足与局限

- **生成器反制**：如果生成的视频故意模仿真实视频的帧间抖动或保留高频空间细节，会缩小 \(\|v_t^{(p)}\|_2\) 的分离性，导致速度门退化均匀注意力。
- **短片段限制**：当 T < 4 时，patch 速度的时域样本太少，增加 \(\tilde{z}_t\) 和 \(r_t\) 的方差。
- **骨干预训练域不匹配**：如果骨干预训练数据与目标域差异很大，冻结的 patch 令牌可能太弱，此时需要 LoRA 或全微调辅助。
- **CNN 骨架**：V-PVP 在 CNN 上增益较小（+2.9），且文中指出内部卷积已部分聚合，因此针对 CNN 的专门读出设计留作未来工作。
- **实验覆盖不足**：未在更真实的视频传播管道（压缩、重采样、屏幕录制、社交媒体重上传等）下评估，可能高估在真实部署中的鲁棒性。
- **未探讨多窗口聚合**：使用均匀采样的固定长度片段，对场景切换或稀疏伪影视频可能不最优。
- **局限性中提到的“良性利用”风险**：论文声明 V-PVP 应作为决策支持工具而非绝对仲裁者，但未详细讨论负面社会影响或逃避手段的应对。
