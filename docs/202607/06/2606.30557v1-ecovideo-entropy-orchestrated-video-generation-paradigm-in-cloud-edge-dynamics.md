## 摘要
DiT视频生成由于迭代的全帧去噪而具有延迟密集型特点，而先前的云边方法主要依赖静态的步间解耦，无法利用帧间相似性或适应系统动态。我们提出EcoVideo，一种用于动态帧间解耦的熵编排框架：早期自注意力熵为帧选择提供无需训练的逐帧信息密度估计；云端大模型对稀疏的高熵关键帧进行去噪；边缘轻量级模型通过运动感知插值及精化重建剩余帧，以保证时间稳定性。EcoVideo还根据实时带宽和计算可用性调整关键帧预算和边缘精化深度，从而在约束下优化端到端延迟。在代表性DiT视频生成器上的实验表明，在低带宽、计算受限的边缘场景下，质量-效率权衡得到改善，端到端加速最高达2.9倍。代码见 https://github.com/IF-LAB-PKU/EcoVideo。

## Abstract
DiT video generation is latency-intensive due to iterative full-frame denoising, while prior cloud-edge methods largely rely on static inter-step decoupling and cannot leverage inter-frame similarity or adapt to system dynamics. We propose EcoVideo, an entropy-orchestrated framework for dynamic inter-frame decoupling: early-stage self-attention entropy provides a training-free estimate of frame-wise information density for frame selection; a cloud large model denoises sparse high-entropy keyframes; and an edge lightweight model reconstructs the remaining frames via motion-aware interpolation with refinement for temporal stability. EcoVideo further adapts the keyframe budget and edge refinement depth to real-time bandwidth and compute availability, optimizing end-to-end latency under constraints. Experiments on representative DiT video generators show improved quality--efficiency trade-offs and up to 2.9x end-to-end speedup in low-bandwidth, compute-limited edge settings. Code is available at https://github.com/IF-LAB-PKU/EcoVideo.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：基于扩散变换器（DiT）的视频生成模型因迭代式全帧去噪而延迟极高，难以规模化部署。现有云边协同方法（如 HybridSD、EC-Diff）采用静态的**步间解耦**策略：云端大模型执行早期去噪步骤，边缘小模型完成后续步骤。这类方法存在三个关键局限：
  - 未利用帧间冗余：即使相邻帧高度相似，仍逐帧重复去噪，计算浪费严重。
  - 小模型去噪能力不足，易导致纹理闪烁、细节崩溃等时序伪影。
  - 云边环境动态变化（带宽、计算资源波动），静态划分无法自适应，端到端效率不稳定。

- **整体含义**：EcoVideo 将协同视角从**步间解耦**转向**帧间解耦**，利用帧间信息密度差异，仅让云端生成高信息量的“关键帧”，边缘用轻量插值重建其余帧，并动态适应系统状态，从而显著加速并保持生成质量。

## 2. 方法论：核心思想、关键技术细节与流程

### 核心思想
- 使用**训练无关的自注意力熵**在线估计逐帧信息密度，区分高熵（关键帧，包含关键运动/结构变化）和低熵（冗余帧）帧。
- 云端大模型只对少量高熵关键帧去噪，并注入低熵帧的冻结上下文以保持全局时空布局。
- 边缘轻量模型（EcoVFI）基于运动感知的评分机制，对关键帧间隙进行**熵感知的贪婪细化插值**，重建全部帧。
- 系统调度器根据实时带宽和计算资源，优化关键帧预算 \(K\) 和插值深度 \(D\)，实现端到端延迟-质量帕累托最优。

### 关键技术细节（文字说明）
1. **帧级注意力熵分析**
   - 在扩散前 10% 步（预热阶段），计算每帧 token 自注意力熵的平均值作为该帧信息密度。
   - 使用指数移动平均（EMA）跨步骤融合，得到稳定熵值 \(e_f\)。

2. **熵驱动的帧间解耦与模型协作**
   - 强制首帧和末帧为关键帧；其余关键帧按熵排序选取 Top \(K-2\)。
   - 云端仅对关键帧 latent 继续去噪；非关键帧 latent 作为**冻结上下文 token** 注入到注意力计算中（停止梯度），保留全局时序信息。

3. **信息重用与帧间插值（EcoVFI）**
   - 对相邻关键帧区间 \((I_a, I_b)\)，计算综合难度分数 \(g(I_a, I_b)\)，融合四个归一化线索：光流运动、熵差、语义结构特征距离（DINO-small）、像素外观差异。
   - 采用**贪婪区间细分**：每次选择最高分区间，插入中间帧（用 EDEN 插值），更新分数，重复直到帧数达到目标长度。插值预算集中在高动态区域。

4. **云边动态感知的适应**
   - 通过在线 profiling 维护单位统计量（云每关键帧时间 \(\bar{c}_c\)、传输载荷 \(\bar{s}\)、边缘每插值帧时间 \(\bar{c}_e(D)\)）。
   - 建立延迟模型 \(T = T_\text{warm} + \frac{K\bar{c}_c}{P_c} + \frac{K\bar{s}}{B} + \frac{(F-K)\bar{c}_e(D)}{P_e}\)。
   - 在候选集合 \(S_K\) 和 \(S_D\) 中最大化质量-延迟效用函数 \((K^*, D^*) = \arg\max [Q(K,D) - \lambda \hat{T}]\)，搜索开销可忽略。

## 6. 主要结论与发现

1. **帧间解耦范式优于步间解耦**：EcoVideo 在所有测试模型上均获得更高的 VBench 总体分（平均比最佳步间基线高 0.084），同时显著降低延迟。
2. **显著效率提升**：在低带宽、计算受限的边缘场景下，端到端加速比最高达 **2.9×**；通信量从步间方法的 17.23 MB 降至 1.10 MB 或更低。
3. **有效抑制时序伪影**：相比 HybridSD 和 EC-Diff，EcoVideo 显著减少了纹理闪烁和细节崩溃，保持视觉连续性。
4. **系统自适应有效**：在不同资源瓶颈下（云算力受限、边缘算力受限、带宽降低），EcoVideo 均保持正加速（最低 1.48×），而固定基线在边缘受限时甚至低于云端直推（0.69×~0.84×）。

## 7. 优点

- **创新思想**：首次将云边视频生成协同从步间解耦转移到帧间解耦，利用帧间信息密度差异，更符合视频时序冗余特性。
- **训练无关性**：注意力熵估计无需额外训练，即插即用，易于集成到现有 DiT 生成器中。
- **自适应能力**：基于实时资源状态动态调整关键帧预算和插值深度，适应真实云边环境波动。
- **综合效果**：同时改善了质量（时序一致性）和效率（延迟/通信），且与现有加速技术（并行、缓存、稀疏注意力）正交。
- **开源代码**：提供完整实现，促进复现和扩展。
