## 摘要
自回归视频扩散模型支持实时合成，但在长时程生成中面临误差累积和上下文丢失的问题。我们发现，自回归视频扩散 Transformer 中的注意力头在功能上扮演着不同的角色：用于细节细化的局部头、用于结构稳定的锚点头以及用于长程上下文聚合的记忆头。然而，现有方法对它们进行统一处理，导致 KV 缓存分配效率低下。我们提出了 Head Forcing，这是一个无需训练的框架，为每种类型的注意力头分配定制的 KV 缓存策略：局部头和锚点头仅保留必要的 token，而记忆头则采用具有动态情节更新的层次化记忆系统，以确保长程一致性。此外，一种按头进行的 RoPE 重编码方案进一步确保了位置编码保持在预训练范围内。在无需额外训练的情况下，Head Forcing 将生成时长从 5 秒延长至分钟级，支持多提示词交互式合成，并持续优于现有基准。项目主页：https://jiahaotian-sjtu.github.io/headforcing.github.io/。

## Abstract
Autoregressive video diffusion models support real-time synthesis but suffer from error accumulation and context loss over long horizons. We discover that attention heads in AR video diffusion transformers serve functionally distinct roles as local heads for detail refinement, anchor heads for structural stabilization, and memory heads for long-range context aggregation, yet existing methods treat them uniformly, leading to suboptimal KV cache allocation. We propose Head Forcing, a training-free framework that assigns each head type a tailored KV cache strategy: local and anchor heads retain only essential tokens, while memory heads employ a hierarchical memory system with dynamic episodic updates for long-range consistency. A head-wise RoPE re-encoding scheme further ensures positional encodings remain within the pretrained range. Without additional training, Head Forcing extends generation from 5 seconds to minute-level duration, supports multi-prompt interactive synthesis, and consistently outperforms existing baselines. *   **核心问题**：自回归（AR）视频扩散模型虽然在实时生成和长视频扩展上具有潜力，但在生成长序列时面临两大挑战：**误差累积**（导致颜色漂移和细节丢失）和**上下文丢失**（由于 KV Cache 容量限制导致身份漂移和场景遗忘）。
*   **研究动机**：作者发现现有的长视频生成方法（无论是训练型还是免训练型）都对 Transformer 中的注意力头（Attention Heads）采取“一视同仁”的统一处理方式。这导致了 KV Cache 分配的低效：不依赖历史信息的头浪费了空间，而需要长程记忆的头却缺乏足够的上下文。

### 2. 方法论：核心思想与关键技术
**核心思想**：通过离线分析识别注意力头的异质功能，并为不同类型的头分配定制化的 KV Cache 策略和位置编码方案。

*   **注意力头分类（Profiling）**：
    *   **局部头（Local Heads）**：主要关注当前帧及邻近帧，负责细节细化。
    *   **锚点头（Anchor Heads）**：对视频首帧（Sink Frame）表现出极高的注意力，负责稳定全局结构。
    *   **记忆头（Memory Heads）**：注意力分布广泛，负责聚合长程上下文。
*   **定制化 KV Cache 分配**：
    *   **局部/锚点剪枝**：局部头仅保留最近帧；锚点头保留首帧+最近帧。这释放了大量缓存空间。
    *   **分层记忆系统（针对记忆头）**：
        *   **快速记忆（Fast Memory）**：保留最近的滑动窗口帧。
        *   **情节记忆（Episodic Memory）**：存储代表性的历史帧。
        *   **动态更新**：基于“场景新颖度”（Key 嵌入的余弦相似度）决定是否存入情节记忆。
        *   **提示词引导压缩**：当情节记忆满时，利用当前文本提示词的对齐分数，筛选并压缩冗余 Token。
*   **按头 RoPE 重编码（Head-wise RoPE Re-encoding）**：
    *   针对不同头缓存长度不一的问题，重新映射帧索引，确保相对位置编码始终处于预训练范围内，防止位置编码越界（OOD）导致的伪影。
